# 모델 해설 — P8~P15 인코더 (채점표 재료)

> ⚠️ **여기 설명하는 P8~P15는 메인 모델 비교에서 제외된 "채점표 재료"다** — 이 중 4개(P8/P10/P11/P12)를
> RRF로 합쳐 채점표(정답 기준)를 만든다. 메인 모델은 P7 하나이며, 비교는 [MODEL_CATALOG](../MODEL_CATALOG.md) 참조.
>
> 인코더 토너먼트에 투입된 모델들이 각각 **무엇이고, 어떻게 만들어졌고, 왜 뽑혔는지**를 설명하는 문서.
> 진행 상황·결과는 [ENCODER_TOURNAMENT_STATUS.md](ENCODER_TOURNAMENT_STATUS.md) 참조.

모든 모델은 동일한 역할을 맡는다: 문서와 후보 키프레이즈를 함께 읽고 **"이 구절이 이 문서의
키프레이즈로 얼마나 적절한가"** 를 점수 하나로 출력하는 랭커의 두뇌(인코더). 두뇌만 바꾸고
나머지(학습 데이터·손실·후보·평가)는 전부 고정했으므로, 성능 차이는 온전히 인코더의 성격 차이다.

---

## P8 — SPECTER2 (`allenai/specter2_base`, 110M)

**출신**: Allen Institute for AI (SciBERT를 만든 곳), 2023년.

**무엇인가**: 과학 논문의 "관계"를 아는 인코더다. 학습 아이디어가 독특한데, 텍스트가 아니라
**인용 그래프**를 신호로 쓴다 — "논문 A가 논문 B를 인용했다면 두 논문의 표현(임베딩)은
가까워야 하고, 인용 관계가 없는 논문 C와는 멀어야 한다"는 삼중항(triplet) 대조학습으로
수백만 편의 논문 위에서 훈련됐다. 원래 용도는 논문 추천·유사 논문 검색 같은
**문서 수준** 과제다.

**P7 SciBERT와의 차이**: SciBERT는 과학 논문의 **어휘와 문장**을 아는 모델(일반 BERT처럼
빈칸 채우기로 학습하되 코퍼스가 과학 논문 114만 편 + 전용 과학 어휘사전 scivocab),
SPECTER2는 과학 논문의 **관계·주제 구조**를 아는 모델이다. 같은 "과학 특화"라도 결이 다르다.

**왜 뽑았나**: "같은 과학 축의 4년 신형이 구형을 이기는가"를 묻기 위해. 아키텍처와 크기가
SciBERT와 같아(BERT-base, 110M) 가장 통제된 비교가 된다.

**실측 결과 미리보기**: F1@5 0.2818로 SciBERT(0.2850)에 근소 열세 — 문서-구절 관련도
과제에서는 문서 수준 인용 신호보다 어휘 수준 이해가 더 유효했다는 시사.

## P9 — DeBERTa-v3-base (`microsoft/deberta-v3-base`, 184M)

**출신**: Microsoft Research, 2021년.

**무엇인가**: 도메인 특화가 전혀 없는 대신 **아키텍처 개선으로 승부**하는 모델이다.
핵심 발명은 두 가지다. ① **분리 어텐션(disentangled attention)** — 보통 BERT는 단어의
'내용'과 '위치' 정보를 하나의 벡터에 섞는데, DeBERTa는 둘을 분리해 각각 어텐션을 계산한다
("무엇이"와 "어디에"를 따로 따져본다). ② **RTD 학습**(v3부터) — 빈칸 채우기 대신,
생성기가 바꿔치기한 가짜 토큰을 판별기가 찾아내는 ELECTRA식 학습으로 효율을 높였다.
어휘사전이 128k로 커서 백본 86M + 임베딩까지 총 184M이다.

수년간 base급에서 분류·랭킹 헤드를 얹는 과제의 최강자로 통했다.

**왜 뽑았나**: **대조군**이다. "과학 특화 없이 그냥 좋은 인코더면 충분한가?"에 답한다.
P9가 P7을 이기면 도메인 특화 서사가 흔들리고, 지면 특화의 가치가 강화된다.

**사고 기록**: 1차 실행에서 loss가 NaN으로 발산했다. 분리 어텐션의 중간 계산값이
half-precision(bf16) 범위를 넘는 **알려진 수치 불안정 이슈**로, 학습률 인하(2e-5→1e-5)와
gradient clipping을 걸어 재실행 중이다.

## P10 — gte-reranker-modernbert-base (`Alibaba-NLP/...`, 149M)

**출신**: 백본은 Answer.AI/LightOn의 ModernBERT(2024), 리랭킹 튜닝은 Alibaba GTE 팀.

**무엇인가**: 두 층으로 이해해야 한다.

- **백본 ModernBERT**: "2018년 BERT를 2024년 기술로 처음부터 다시 지으면?"이라는 프로젝트의
  산물. 회전 위치 인코딩(RoPE), 지역-전역 교대 어텐션, FlashAttention 등 LLM 시대의
  기법들을 인코더에 이식해 **8,192토큰 문맥**과 빠른 속도를 얻었다 (BERT는 512토큰).
- **GTE 리랭커 튜닝**: 그 백본을 대규모 질의-문서 관련성 데이터로 **cross-encoder 리랭킹
  과제 자체를 사전학습**시킨 버전. 즉 우리가 시키려는 일(쌍을 읽고 관련도 점수 내기)을
  이미 수백만 번 연습하고 입장하는 선수다.

**왜 뽑았나**: 2026년 리랭킹 벤치마크에서 자기보다 8배 큰 모델과 동급 정확도를 내는
최상위권으로 확인된 모델(웹 검증 완료). "현세대 아키텍처 + 랭킹 조기교육"의 조합이
도메인 특화를 이기는지 본다. **이번 토너먼트의 우승 후보.**

---

## P11 — Qwen3-Reranker-0.6B (`Qwen/Qwen3-Reranker-0.6B`, 596M)

**출신**: Alibaba Qwen 팀, 2025년.

**무엇인가**: 로스터에서 유일하게 **출신 성분이 다르다**. 나머지는 모두 문장을 양방향으로
읽는 인코더(BERT류)인데, 이것은 **LLM(디코더)을 리랭커로 개조**한 것이다. Qwen3 0.6B
언어모델에게 "이 문서가 이 질의와 관련 있나?"를 지시문으로 묻고 "yes"라고 답할 확률을
점수로 쓰도록 대규모 다국어 검색 데이터로 훈련했다. 2025~26년 MTEB 리랭킹 부문의
표준 계열이다.

**우리 실험에서의 개조**: 디코더라 [CLS] 토큰이 없어 **마지막 토큰의 은닉 상태**를 문장
표현으로 쓰고(인과 어텐션에서 마지막 토큰만이 전체를 본다), 그 위에 우리 MLP 헤드를 얹는다.
596M으로 로스터 최대라 배치를 절반으로 줄이고 gradient checkpointing을 켠다.

**왜 뽑았나**: "LLM의 언어 이해력이 fine-tuning 예산 10만 쌍 규모에서도 인코더들을
이기는가" — LLM 혈통 축의 대표. (원래 이 자리에 EmbeddingGemma를 검토했으나 그건
bi-encoder 임베딩 특화라 cross-encoder 과제엔 오프라벨이어서, 랭킹으로 검증된 Qwen3로
교체하고 Gemma는 P12로 별도 편성했다.)

## P12 — EmbeddingGemma-300m (`google/embeddinggemma-300m`, 303M)

**출신**: Google DeepMind, 2025년. Gemma 3 계열.

**무엇인가**: Google이 Gemma 3 언어모델을 바탕으로 만든 **경량 임베딩 전용 모델**이다.
문장·문서를 벡터 하나로 압축하는 bi-encoder로 설계됐고, 출시 당시 500M 미만급 다국어
임베딩 벤치마크(MTEB) 최상위였다. 온디바이스(휴대폰) 검색을 겨냥한 작고 빠른 모델.

**우리 실험에서의 개조와 주의**: 이 모델의 본업은 "문장 → 벡터"이지 "쌍 → 점수"가 아니다.
우리는 문서+구절을 이어 넣고 **mean pooling**(전 토큰 평균)으로 표현을 뽑아 헤드를 얹는
**오프라벨 사용**을 한다 — 그래서 성적이 낮게 나와도 "Gemma가 나쁘다"가 아니라
"임베딩 특화 모델을 cross-encoder로 전용하면 손해"로 해석해야 공정하다.

**왜 뽑았나**: 사용자 지정(Gemma 계열 포함 요청) + "bi-encoder 혈통 vs cross-encoder
혈통"이라는 비교 축이 하나 더 생긴다. 참고로 HF에서 라이선스 게이트가 걸린 모델이라
토큰 권한 설정이 필요했다 (해결 완료).

## P14 — SciDeBERTa-CS (`KISTI-AI/scideberta-cs`, ~180M)

**출신**: KISTI(한국과학기술정보연구원), DeBERTa 기반.

**무엇인가**: DeBERTa를 **컴퓨터과학 논문 코퍼스로 도메인 적응(continued pretraining)**
시킨 모델이다. 일반 코퍼스로 다 배운 모델에게 CS 논문만 추가로 읽혀서 CS 어휘·표현에
익숙하게 만든 것.

**왜 뽑았나**: 두 가지 이유로 가치가 크다. ① KP20k가 바로 CS 논문 데이터셋이라
**도메인이 정확히 일치**하는 유일한 DeBERTa다. ② **P9(무특화 DeBERTa)와의 짝 비교**가
성립한다 — 같은 계열에서 "CS 적응을 했다/안 했다"만 다르므로, 이 짝의 점수 차이는
도메인 적응 사전학습(DAPT)의 순수 효과를 보여준다. (단 P9는 v3, 이것은 v2 세대라
완전히 순수하지는 않음을 명시한다.)

## P15 — CS-RoBERTa (`allenai/cs_roberta_base`, 125M)

**출신**: Allen AI, 2020년. 유명 논문 *"Don't Stop Pretraining"* 의 공식 산출물.

**무엇인가**: RoBERTa(BERT의 개선판 — 더 오래, 더 많은 데이터로, 더 잘 튜닝해 학습한 버전)를
**S2ORC의 CS 논문 220만 편으로 계속 사전학습**한 모델. "범용 모델도 목표 도메인 텍스트를
더 읽히면 그 도메인 과제가 좋아진다"는 DAPT 가설을 입증한 바로 그 실험의 모델이다.

**왜 뽑았나**: P14와 함께 **CS 도메인 정확 일치** 조를 이룬다. RoBERTa 혈통이라
P7(BERT)·P9(DeBERTa)와 또 다른 오류 패턴이 기대되고, 이 다양성이 P13 앙상블의 재료가 된다.

## P13 — RRF 앙상블 (모델이 아니라 결합 규칙)

**무엇인가**: 새 모델이 아니다. 위 랭커들이 같은 후보에 매긴 **순위**를
$score(c)=\sum_{m} 1/(60+rank_m(c))$ 로 합산하는 Reciprocal Rank Fusion이다.
점수 스케일이 제각각(logit, 0~1, RRF)이어도 순위만 쓰므로 보정이 필요 없다.

**멤버 선발**: {P2 fusion(비학습 — 오류 패턴이 가장 달라 다양성 담당), P7~P12, P14, P15}
후보 중 2~4개 조합을 전수 탐색하되, **test가 아니라 validation 홀드아웃 1,000편**에서
고른다 (test에서 고르면 선택 편향 — λ 튜닝 때 확립한 규약).

**기대치**: P7이 이미 생성확률·출처 신호를 aux 피처로 흡수했으므로 이득이 제한적일 수
있다. +0.005 미만이면 "단독 모델이 앙상블 이득을 내재화했다"는 결론 — 그 자체로 발견이다.

## 부록 — 혈통 지도와 로스터 설계 논리

```text
BERT 혈통 (과학)   : P7 SciBERT(어휘) ←→ P8 SPECTER2(인용)      … 같은 축, 다른 신호
DeBERTa 혈통       : P9 무특화 ←→ P14 CS 적응                    … DAPT 효과 짝 비교
RoBERTa 혈통       : P15 CS 적응
현세대 인코더      : P10 ModernBERT + 리랭킹 사전학습             … 우승 후보
LLM 디코더 혈통    : P11 Qwen3-Reranker(랭킹 특화) ←→ P12 Gemma(임베딩 특화)
비학습 수식        : P2 score fusion                              … 앙상블 다양성 담당
```

여섯 혈통 × 특화 유무가 겹치지 않게 배치되어, 누가 이기든 "도메인 특화 덕 / 아키텍처 덕 /
랭킹 사전학습 덕 / 크기 덕" 중 어느 것인지 좁혀 해석할 수 있다 — 이것이 로스터가
"강한 순"이 아니라 "축이 다른 순"으로 짜인 이유다.
