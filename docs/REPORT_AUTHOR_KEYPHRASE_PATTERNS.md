# 보고서 — 저자들은 키프레이즈를 어떻게 고르는가 (KP20k 전수 패턴 분석)

> 작성일: 2026-07-14 · 대상: KP20k train **530,809편 전체**, 키프레이즈 2,808,770개
> 데이터: `outputs/metrics/author_patterns.json` · 구현: `scripts/author_pattern_analysis.py`
> Phase B(Qwen LLM 정성 분류)는 실행 경로 확정 후 별도 절로 추가 예정.

---

## 요약 — 저자 선정은 무작위가 아니라 "관행 + 공식"이다

| # | 패턴 | 수치 |
|---|---|---|
| 1 | 개수 관행 | 평균 5.29개, **83%가 3~6개** |
| 2 | 길이 관행 | 평균 1.94단어 — **2단어가 46%** (bigram 지배) |

| 6 | 배치 관행 | absent(R/M/U)는 목록 **뒤쪽**(상대위치 0.76)에 배치 |

## 패턴 1·2 — "다섯 개, 두 단어씩"이라는 불문율

- 문서당 개수: 평균 5.29, 중앙값 5. **83.2%가 3~6개 구간** — 학회/저널 투고 양식의
  "3~6 keywords" 지침이 그대로 각인된 분포다.
- 길이: 1단어 33.6% / **2단어 46.3%** / 3단어+ 20.2%. 저자들의 기본 단위는
  `neural network`, `data mining` 같은 **2단어 복합명사**다.

**시스템 시사점**: 우리 후보 필터(1~5단어)와 평가 K=5가 이 관행과 정합함을 재확인.

## 패턴 3 — 저자는 자기 제목에서 베낀다

- 키프레이즈의 **27.8%가 제목 안에** 그대로 등장한다. 제목이 ~10단어에 불과함을 감안하면
  단위 길이당 밀도는 초록의 수십 배 — **제목은 저자 스스로 고른 키프레이즈 후보 1순위**다.
- 초록 첫 문장 재등장도 23.4% — "문서 앞부분 편향"이 실재한다.

**시스템 시사점**: 우리 score fusion의 title-overlap 항(w=0.10)과 입력에 제목을 반드시
포함하는 설계(플랜 수정1)가 데이터로 정당화된다. 빈도 베이스라인이 "앞부분 보너스"만으로
absent R@10에서 선전했던 것(2차 노트북 Rank CI)도 같은 원리.

## 패턴 4 — 관행어 공유 풀: "paper"도 키프레이즈다

고유 정규형 645,406개 중 극소수가 반복 사용된다:

- **top-100 구절 = 전체 인스턴스의 10.4%**, top-1000 = **29.3%**
- 최다 사용 구절(어간형): `algorithm`(10,483회), **`paper`(8,776회)**, `perform`(7,813),
  `design`(7,456), `model`(7,133), `optim`(6,995), `genet algorithm`(6,838), `simul`(6,797) …

`paper`가 키프레이즈로 8,776번 달렸다는 것은 이 목록이 "문서의 핵심 개념"만이 아니라
**ACM 분류어·투고 관행의 산물**이기도 함을 보여준다. 상위 20개가 거의 전부
방법 불문의 일반어다.

**시스템 시사점 (중요)**: 이것이 **P6(PRMU-가중 pairwise)이 신규 논문에서 `performance`,
`experimentation` 같은 일반어를 승격시킨 근본 원인**이다. exact-match 학습은 이 관행 풀을
"정답 분포"로 흡수한다 — 지표 최적화와 유용성의 괴리(우리의 반복 발견)가 데이터 기원까지
추적된 셈이다.

## 패턴 5 — 구성 공식: "관행어 한 줌 + 내 전문어 한 줌"

문서 단위로 보면 (20% 표본, n≈106k):

- 문서당 **단골어(전체 1,000회+) 평균 0.8개** — 34.3%의 문서가 1개 이상 포함
- 문서당 **희귀어(전체 5회 이하) 평균 1.58개** — 이 논문에서만 쓰는 고유 전문어
- 35.7%는 단골어 없이 전문어 위주로만 구성

즉 전형적 레시피는 **"분야 표시용 일반어 ~1개 + 논문 고유 전문어 ~1.6개 + 중간 빈도
주제어 ~2.7개"** — 저자는 "분류되기 위한 태그"와 "구별되기 위한 태그"를 섞는다.

**시스템 시사점**: 단일 랭킹 함수가 이 두 이질적 목적(분류용 일반어 vs 구별용 전문어)을
동시에 맞히려니 긴장이 생긴다 — 색인 실험에서 일반어가 검색에 무용했던 것과 연결하면,
"무엇을 재현할 것인가"는 응용이 결정해야 한다는 결론이 다시 나온다.

## 패턴 6 — 축약어와 배치 순서

- 단일 토큰 축약어(svm, qos, lstm류) 추정 **9.5%** — 열 개 중 하나는 두문자어.
- **absent(R/M/U)의 목록 내 평균 상대위치 0.756** — 저자들은 본문에 있는 용어를 앞에,
  추상 개념·상위어를 **뒤에** 단다. (P를 등장순으로 정렬해 저장했다는 데이터 카드 설명과
  정합 — 그래서 "목록 순서 = 중요도"가 아니라는 플랜 수정5가 재확인된다.)

## 보너스 발견 — 팀 통계 미스터리의 해소

전수 스캔에서 공식 `prmu` 필드의 P 비율은 **60.78%** 로 나왔다. 이는 마스터 플랜 2.3절이
"공식 카드(58.2%)와 다르다"고 지적했던 **팀의 자체 계산값 60.78%와 정확히 일치**한다.
즉 팀 계산은 틀리지 않았다 — 공식 필드를 인스턴스 단위(micro)로 합산하면 60.78%가 나오고,
데이터 카드의 58.2%는 집계 방식(문서별 평균 등)이 다른 것으로 보인다. 보고서에는
"공식 prmu 필드 기준 micro 60.8% / 카드 표기 58.2% (집계 방식 차이)"로 병기하면 된다.

## Phase B 예고 — Qwen LLM 정성 분류 (경로 확정 대기)

정량 분석이 못 보는 것: `neural network`가 이 논문에서 **방법**인지 **연구 대상**인지,
일반어가 "분야 태그" 역할인지. 표본 300~500편의 키프레이즈를 Qwen에게
{분야명 / 방법·기법 / 응용 도메인 / 시스템·축약어 / 관행어} 로 분류시켜
패턴 5의 "구성 공식"을 의미 범주로 확장할 예정. 실행 경로(외부 API / 토너먼트 후
로컬 GPU / CPU 소형) 확정 후 이 문서에 절을 추가한다.

## 한 줄 결론

> KP20k 저자들의 키프레이즈 선정은 **"3~6개·주로 2단어·제목에서 28% 재활용·공유 관행어
> 29%·일반어 1개+전문어 1.6개 혼합·absent는 뒤에"** 라는 강한 관행 구조를 따른다.
> 이 관행이 곧 exact-match 학습 신호가 되므로, 관행(일반어 포함)을 재현하는 모델이
> F1은 높지만 검색 유용성은 낮아지는 — 본 연구의 핵심 긴장 — 데이터적 기원이 여기에 있다.
