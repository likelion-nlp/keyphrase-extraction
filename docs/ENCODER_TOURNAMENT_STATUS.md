## 0. 한 줄 목적과 한 줄 결론

**목적**: P7(SciBERT+aux, F1@5 0.2850)의 힘이 인코더에서 오는지 방법론에서 오는지 —
레시피(aux 3피처 + PRMU-가중 pairwise, margin 0.3, 학습쌍 105,464개, 후보 pool,
test 20,000, top-10)를 완전히 고정하고 **인코더만** 교체해 판정.

**결론**: BERT급(110~150M)에서는 인코더를 바꿔도 F1@5가 0.2818~0.2850(폭 0.0032)로
사실상 불변 — **성능의 원천은 방법론**. 규모를 4배 키운 디코더 혈통(P11 Qwen3 0.6B)만
+0.0076 우위(단독 최고). 이 인코더들의 RRF 합의는 **F1 참가 모델이 아니라 채점표(정답 기준)**로
쓰여, P7·베이스라인의 랭킹을 채점한다 (§3c).

- [x] **인코더 불감 판정**: BERT급 4종(P7/P8/P10/P15) 폭 0.0032 < 기준 ±0.005 →
  **방법론 기여로 결론**. 2019년 SciBERT를 2026년 리랭킹 특화 모델로 바꿔도 안 오른다.
- [x] **규모·혈통 효과**: P11(0.6B, 디코더)만 +0.0076 — 유일하게 기준을 넘은 단독 개선.

  (Gemma는 bi-encoder 임베딩용으로 학습돼 cross-encoder 입력에 부적합).

  인코더가 아니라 **PRMU-가중 pairwise 방법론의 성질**로 확정 (P5/P6에 이어 6번째 재현).

  P7·베이스라인을 채점한다 — F1 경쟁이 아니라 채점 기준으로서의 역할(§3c). (앙상블 자체
  F1 수치 P16 0.3011/P13 0.3017은 데이터로만 보존.)

## 3. 채점표(앙상블) 구성 상세

- 구성 프로토콜: validation 홀드아웃 1,000편(학습 10,000편과 불겹침)에서 RRF(1/(60+rank))
  조합 크기 2~4 전수 탐색 → 가장 안정적인 합의 조합 선택. **P12(Gemma)는 사용자 지정 필수 멤버**.
- dev 상위: `P7+P10+P11+P12` = `P7+P11+P12` = 0.3062 동률 1위 — 단독 0.2670인 Gemma도
  혈통이 달라 오답 패턴이 다르므로 합의에 기여(다양성 > 단독 성능).
- **채점표로는 P16 구성(P8/P10/P11/P12) 사용** — P7을 포함하지 않아야 "P7이 채점표를
  재현하나"를 순환 없이 잰다. P13(P7 포함)은 조합 실험 기록으로만 남긴다.
- 앙상블 자체 지표(참고, gold 대비 — 경쟁 순위 아님): P16 F1@5 0.3011 / P13 0.3017.
- 탐색 전체 기록: `outputs/metrics/ensemble_selection.json` (조합별 dev F1, 필수 멤버 표기).

## 3b. P16 — 팀 사양 앙상블 (팀원 요청 버전)

앙상블을 수행한 원래 배경은 **팀원들의 요청**이었다: *"LLM으로 생성한 keyphrase 랭킹
채점표 도입 — 각 모델이 10개 추출 → 중복값 제거 후 리랭킹, 지정 5개 모델(SPECTER2 /
DeBERTa-v3 / gte-reranker-ModernBERT / Qwen3-Reranker-0.6B / Gemma) 앙상블"*.
P13은 이를 일반화(검증 기반 조합 선택 + 풀 전체 RRF)한 것이고, 팀 사양 그대로의
버전을 **P16**으로 별도 구현했다 (`scripts/team_spec_ensemble.py`).

| 구분 | P13 (일반화) | P16 (팀 사양) |
|---|---|---|
| 멤버 | 검증 1,000편이 선택 (P7+P10+P11+P12) | 팀 지정 5개 고정 (P9는 학습 실패로 4개) |
| 후보 | 풀 전체(문서당 ~37개)를 RRF | **각 멤버 top-10 합집합**(평균 13.7개)만 리랭킹 |
| F1@5 | **0.3017** | 0.3011 |
| present F1@5 | 0.3798 | **0.3800** |
| absent R@10 | 0.0506 | 0.0506 (동률) |
| nDCG@10 | **0.4174** | 0.4168 |

**해석**: 두 방식의 차이는 0.0006 — 사실상 동급. 팀 사양(top-10 합집합 후 리랭킹)이
거의 손해가 없다는 것은 각 멤버의 top-10 밖에서 앙상블이 건질 후보가 거의 없다는 뜻
(RRF 상위권은 결국 누군가의 top-10 안에 있다). 팀 보고에는 **사양 그대로인 P16을
공식 수치로, P13을 상한 참고치**로 쓰면 깔끔하다. 유일한 미충족은 P9(DeBERTa-v3)
탈락으로 5개가 아닌 4개 앙상블이라는 점 (bf16 발산 2회 — §5 사고 로그).

## 3c. 앙상블 = 채점표 — P7 vs 베이스라인 채점

앙상블은 gold-F1 경쟁 모델이 아니라 **여러 리랭커의 합의(consensus)로 만든 채점표(정답 기준)**다.
앙상블 순서를 정답으로 두고 "어떤 단일 모델이 채점표를 가장 잘 재현하나"를 채점했다 —
P7(Hybrid) vs KeyBART(베이스라인). (노트북 `05_ensemble_reference_comparison.ipynb`)

| 지표 (앙상블 기준) | P7 | KeyBART | P7 우세 문서 | Wilcoxon p |
|---|---:|---:|---:|---:|
| nDCG@10 | **0.898** | 0.784 | **85.2%** | ≈0 |
| RBO@10 | **0.452** | 0.325 | **85.1%** | ≈0 |

**결론**: P7 단일 모델이 앙상블 합의를 사실상 재현한다(문서 85%에서 베이스라인 압도). 이는
"인코더 불감 → 방법론 기여"(§2) 결론과 정합하며, 4모델 앙상블이 부담될 때 **P7을 배포용
단일 프록시**로 쓸 근거가 된다. 상세: [ENSEMBLE_REFERENCE_COMPARISON.md](ENSEMBLE_REFERENCE_COMPARISON.md).
